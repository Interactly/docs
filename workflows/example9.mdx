## High-Level Description

This workflow introduces two advanced features that enhance user experience and enable cross-node data flow: **Tool Static Messages** and **Tool Result Runtime Variables**. Static messages provide user feedback during tool execution ("Analyzing your symptoms..."), while runtime variables store tool results for use in downstream agents. The example demonstrates a patient triage workflow where symptom assessment results are stored and used by specialized care path agents.

## New Concepts Introduced
Building upon Examples 1-8, this workflow adds:
1. **Tool Static Messages** (static messages **in tools**):
    - Display messages to users while tools execute
    - Provide feedback during potentially long operations
    - Improve perceived responsiveness
    - Set user expectations
2. **Tool Result Runtime Variables** (result_runtime_variable_name):
    - Store tool return values in named variables
    - Access stored values in downstream agent prompts
    - Enable data flow between agents without structured output schemas
    - Use template syntax: [[variable_name.field]]
3. **Edge Static Messages** (static_messages **in edges**):
    - Display messages during transitions between agents
    - Inform users about routing decisions
    - Provide context for workflow progression
4. **Liquid Template Filters in Expressions**:
    - [[variable | upcase]]: Convert to uppercase
    - Enables case-insensitive comparisons 
    - Useful for enum validation

## Tool Configuration with Static Messages

## Tool: assess_patient_symptoms

**Purpose**: Analyze patient symptoms and determine urgency level with user feedback.
### Configuration:
```Json
  {
  "name": "assess_patient_symptoms",
  "signature": "Analyze patient symptoms and determine urgency level and recommended care path",
  "args_schema": {
    "type": "object",
    "properties": {
      "symptoms": {
        "type": "string",
        "description": "Description of patient symptoms"
      },
      "duration": {
        "type": "string",
        "description": "How long the symptoms have been present"
      },
      "severity": {
        "type": "string",
        "enum": ["mild", "moderate", "severe"],
        "description": "Patient's description of severity"
      }
    },
    "required": ["symptoms", "duration", "severity"]
  }
}
```   

### Static Messages Configuration ‚≠ê:
```Json
   {
  "static_messages": [
    "Let me analyze your symptoms...",
    "Evaluating your condition...",
    "Assessing the urgency of your case..."
  ]
}
```
### Result Runtime Variable ‚≠ê:
```Json
result_runtime_variable_name: "patient_assessment_result"
```
### Python Code:
```Json
   def assess_patient_symptoms(symptoms: str, duration: str, severity: str) -> dict:
    '''Assess symptoms and return urgency level with care recommendations'''
    symptoms_lower = symptoms.lower()
    
    # Emergency keywords
    emergency_keywords = [
        "chest pain", "difficulty breathing", "severe bleeding", 
        "stroke", "unconscious", "seizure", "severe headache"
    ]
    
    # Urgent keywords
    urgent_keywords = [
        "fever", "vomiting", "diarrhea", "sprain", "cut",
        "burn", "infection", "pain", "rash"
    ]
    
    is_emergency = any(keyword in symptoms_lower for keyword in emergency_keywords)
    is_urgent = any(keyword in symptoms_lower for keyword in urgent_keywords)
    
    if is_emergency or severity == "severe":
        return {
            "urgency_level": "EMERGENCY",
            "care_path": "emergency_room",
            "wait_time": "immediate",
            "recommendation": "Seek emergency care immediately - call 911 or go to nearest ER",
            "symptoms_summary": symptoms,
            "severity": severity,
            "duration": duration
        }
    elif is_urgent or severity == "moderate":
        return {
            "urgency_level": "URGENT",
            "care_path": "urgent_care",
            "wait_time": "within 24 hours",
            "recommendation": "Visit urgent care or schedule same-day appointment",
            "symptoms_summary": symptoms,
            "severity": severity,
            "duration": duration
        }
    else:
        return {
            "urgency_level": "ROUTINE",
            "care_path": "primary_care",
            "wait_time": "within 1-2 weeks",
            "recommendation": "Schedule routine appointment with your primary care provider",
            "symptoms_summary": symptoms,
            "severity": severity,
            "duration": duration
        }
 ```
### What Happens:
 1. User provides symptoms
 2. Agent calls assess_patient_symptoms()
 3. **User sees**: "Let me analyze your symptoms..." (static message)
 4. Tool executes and returns assessment dict
 5. Result stored in patient_assessment_result runtime variable
 6. Result available to all downstream agents

## Key Takeaways

### Tool Static Messages:
 - Provide real-time feedback during tool execution
 - Improve user experience with progress updates
 - Set expectations ("Analyzing...", "Calculating...")
 - Can include multiple messages for longer operations
 - Displayed in order while tool runs
### Tool Result Runtime Variables:
 - Store tool return values in named variables
 - Access in downstream agent prompts using [[variable_name.field]]
 - Enable data flow without structured output schemas
 - Cleaner than passing data through conversation context
 - Available to all subsequent agents in the workflow
### Edge Static Messages:

 - Display during transitions between agents
 - Inform users about routing decisions
 - Provide context for workflow progression
 - Examples: "Routing to specialist...", "Processing your request..."
### Liquid Template Filters:

 - | upcase: Convert to uppercase
 - | downcase: Convert to lowercase
 - Useful for case-insensitive comparisons
 - Applied in expression conditions
### Benefits:

 - **User Experience**: Transparent, responsive interactions
 - **Data Flow**: Clean cross-agent data sharing
 - **Flexibility**: Tool results accessible anywhere downstream
 - **Debugging**: Easy to trace data flow through runtime variables
 ### Use Cases:

 - **Medical Triage**: Assessment results guide care path
 - **Financial Applications**: Credit checks inform product offerings
 - **Customer Service**: Account lookup results customize responses
 - **E-commerce**: Inventory checks determine availability messaging
 - **Form Validation**: Validation results shown during multi-step processes
### Comparison to Previous Examples:
 - Example 8: Inline tools (no static messages or runtime vars)
 - Example 9: Adds tool static messages AND runtime variables
 - Enables richer user experience and cleaner data flow
### Technical Flow:
```Text
 1. Agent calls tool with arguments
 2. USER SEES: Tool static messages
 3. Tool executes and returns result
 4. Result stored in runtime variable
 5. Conditional edge evaluates using runtime variable
 6. USER SEES: Edge static message (during transition)
 7. Destination agent uses runtime variable in prompt
 8. Destination agent responds with context from tool resul
```
### Best Practices:
 - Use descriptive static messages
 - Store comprehensive tool results
 - Access specific fields with dot notation
 - Use filters for robust comparisons
 - Provide edge messages for clarity
 
This workflow demonstrates how static messages and runtime variables create seamless, context-aware conversational experiences while maintaining clean data flow across agents.
